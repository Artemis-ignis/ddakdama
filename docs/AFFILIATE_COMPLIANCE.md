# 제휴 링크 공개 배포 점검

> 이 문서는 구현상의 안전장치와 공개 배포 위험을 정리한 개발 체크리스트입니다. 법률 자문이나 플랫폼의 사전 승인을 대신하지 않습니다.

## 현재 v0.2 구현

- 파트너스 기능 기본 꺼짐
- 설정 화면의 명시적 고지
- 사용자가 토글을 켤 때 확인창 표시
- 후보 카드에 파트너스 적용 여부 표시
- 최종 일괄 담기 확인창에 적용 상품 수 표시
- 사용자가 후보 검색·상품 링크 클릭·일괄 담기 버튼을 누른 경우에만 딥링크 생성·사용
- 브라우저의 기존 쿠팡 URL이나 쿠키를 상시·무단으로 교체하지 않음
- 파트너스 실패 상품은 일반 링크로 처리
- Secret Key는 서버에만 보관

## Chrome Web Store 핵심 위험

Chrome Web Store는 제휴 프로그램의 사전·UI·스토어 고지, 관련 사용자 동작, 그리고 제휴 링크가 적용되는 순간의 직접적이고 투명한 사용자 혜택을 요구합니다. 단순히 운영자가 수수료를 얻는 구조만으로는 심사에 충분하지 않을 수 있습니다.

따라서 **현재 파트너스 기능이 들어간 그대로 공개 스토어 승인을 보장할 수 없습니다.** 공개 배포 방식은 아래 중 하나로 정하는 것이 안전합니다.

### 선택 A: 스토어 버전에서 파트너스 비활성화

- Chrome Web Store 빌드는 일반 링크만 사용
- 파트너스 기능은 자체 배포·사내 배포·비공개 테스트에서만 사용
- 가장 단순하지만 수익화는 별도 웹 서비스나 명시적 광고 링크로 분리

### 선택 B: 실제 사용자 혜택 추가

예:

- 유효한 쿠폰·할인이 발견된 상품에만 제휴 링크 적용
- 수수료 일부를 사용자에게 캐시백
- 사용자가 선택한 공익단체에 실제 기부하고 내역 공개
- 제휴 링크 적용 시 제공되는 명확한 유료 기능 할인

혜택이 실제로 제공되고, 링크 적용과 직접 연결되며, UI·스토어 설명·약관에서 검증 가능해야 합니다.

### 선택 C: 공개 전 플랫폼 사전 문의

- 확장 프로그램의 정확한 동작 영상을 준비
- 사용자 동작, 고지 위치, 링크 적용 시점, 사용자 혜택을 설명
- Chrome Web Store 개발자 지원에 사전 문의
- 쿠팡 파트너스에도 브라우저 확장 프로그램이 허용 매체인지 서면 확인

## 공개 스토어 고지 초안

```text
딱담아는 선택적으로 제휴 링크를 사용할 수 있습니다. 제휴 모드를 사용하면 일부 상품 링크를 통해 운영자에게 수수료가 지급될 수 있습니다. 제휴 링크는 사용자의 명시적 동작과 동의 후에만 적용되며, 적용 여부는 상품마다 표시됩니다. 지원되지 않는 상품은 일반 링크로 처리됩니다.
```

이 문구만으로 정책 충족이 보장되는 것은 아닙니다. 실제 사용자 혜택과 동작이 일치해야 합니다.

## 쿠팡·파트너스 허용 여부 점검

쿠팡의 일반 서비스 이용정책은 회사가 제공하지 않은 비정상적인 방법으로 시스템에 접근·이용하는 행위와 회사 동의 없는 영리 목적 이용을 금지하는 조항을 포함합니다. 브라우저 확장 프로그램이 사용자의 명시적 동작을 대신해 화면을 읽고 장바구니 버튼을 누르는 방식이 허용되는지는 공개 문서만으로 확정하기 어렵습니다. **상용 출시 전에는 쿠팡과 쿠팡 파트너스에 기능 설명·동작 영상을 보내 서면 허용을 받는 것이 안전합니다.** 허용을 받기 전에는 내부 테스트·개인용 MVP로 제한하세요.

공개 전 계정의 최신 약관·운영정책에서 다음을 확인합니다.

- 브라우저 확장 프로그램·자동화 도구가 허용 홍보 매체인지
- API 딥링크 사용 조건
- 자동 리디렉션이나 백그라운드 탭 경유 허용 여부
- 필수 경제적 이해관계 고지 문구와 위치
- 자기구매, 가족 구매, 리워드, 캐시백 정책
- 클릭 조작·자동 클릭·비정상 트래픽 기준
- 상표·로고·상품 이미지 사용 조건

명확하지 않으면 파트너스 기능은 비활성화한 채 공개하고 서면 답변을 받은 뒤 업데이트합니다.

## 로그·통계 원칙

- 제휴 링크 생성 횟수와 성공률은 익명 집계 가능
- 상품 URL 전체, 사용자 목록, 서버 토큰을 장기 보관하지 않기
- 클릭 수를 인위적으로 늘리는 자동 새로고침·반복 경유 금지
- 사용자가 일괄 담기를 한 번 눌렀을 때 필요한 상품 수만 처리
- 실패 재시도에 상한과 지연 적용
