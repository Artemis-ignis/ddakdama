# 딱담아 Production Beta v1.0.2 아키텍처

## 구성

- `apps/extension`: 사용자가 설치하는 단 하나의 Chrome Manifest V3 확장 프로그램
- `apps/server`: OpenAI Apps SDK/MCP, 페어링·handoff, 쿠팡 파트너스 서버
- `packages/core`: 목록 파서, 수량 계획, 장바구니 결과 요약을 공유하는 순수 TypeScript 패키지

## 사용자 흐름

1. 확장 프로그램 또는 ChatGPT 앱에서 한국어 쇼핑 목록을 입력합니다.
2. `packages/core`가 용량, 함량, 포장 규격과 요청 실물수량을 분리합니다.
3. 확장 프로그램이 사용자의 명시적 검색 동작 후 쿠팡 검색 결과를 읽습니다.
4. 후보별 상품명, 가격, 묶음과 배송 정보를 비교하고 사용자가 후보를 바꿀 수 있습니다.
5. 가격이 없거나 수량이 정확히 나누어지지 않는 후보는 자동 담기에서 제외합니다.
6. 첫 번째 담기 버튼은 사전검사 결과만 보여주고, 두 번째 명시적 승인 후에만 장바구니를 변경합니다.
7. 각 상품은 담기 전후 `productId + vendorItemId + itemId` 복합 SKU 수량 차이를 비교합니다. 일부만 성공하면 `PARTIAL_FAILURE`입니다.
8. 중단된 작업은 `chrome.storage.local` journal에서 복원하고, 현재 장바구니 수량을 다시 읽어 목표 수량까지 남은 수량만 처리합니다.
9. 결과 화면은 상세 확인가, 장바구니 표시가와 차액을 구분하며 가격을 확인하지 못한 상품은 자동으로 담지 않습니다.
8. 결제와 주문 확정은 자동화하지 않습니다.

## GPT 앱 연결

확장 프로그램이 일회용 6자리 코드와 기기 토큰을 발급받습니다. 사용자가 ChatGPT 위젯에 코드를 입력하면 해당 기기 ID가 연결됩니다. `send_cart_plan`은 짧은 수명의 handoff를 만들고, 기기 토큰을 가진 확장 프로그램만 이를 가져와 ACK할 수 있습니다. 토큰·서버 URL·내부 ID는 일반 사용자 화면에 표시하지 않습니다.

## 쿠팡 연동

파트너스 키가 있으면 서버의 공식 Product Search와 Deep Link 어댑터를 사용할 수 있습니다. 키가 없거나 API 결과가 부족하면 확장 프로그램의 일반 쿠팡 검색으로 계속 진행합니다. DOM 선택자는 `coupang-content.ts` 한 곳에 모으며 CAPTCHA와 보안 확인을 우회하지 않습니다.

## 데이터 경계

- 쿠팡 로그인 쿠키, 비밀번호, 카드정보는 읽거나 서버로 보내지 않습니다.
- Access Key와 Secret Key는 서버 환경변수에만 둡니다.
- 기기 토큰은 해시로 조회하며 handoff에는 TTL과 ACK를 적용합니다.
- 현재 로컬 단일 프로세스 저장소는 개발·개인 사용용입니다. 다중 사용자 운영 전에는 TTL 저장소로 교체해야 합니다.
