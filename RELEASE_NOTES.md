# 딱담아 v1.0.0 Production Beta

## 핵심 기능

- 자연어 쇼핑 목록을 브랜드·제품명·용량·함량·포장 규격·요청 실물수량으로 분리
- 고정 목록 5종·실물 7개를 정확히 해석하고 `100mg 240정`을 1병으로 처리
- 쿠팡 후보의 브랜드·핵심 제품명·규격·묶음·가격을 엄격하게 검증
- 단품과 묶음 상품을 고려한 정확한 장바구니 구매수량 계산
- 상세페이지 확인가가 없는 상품, 품절, 필수 옵션, 규격 불일치 상품 자동 담기 차단
- 사전검사 문제를 숨기지 않고 전체 취소·문제 확인·가능한 상품만 담기를 명시적으로 선택
- productId·vendorItemId·itemId 복합 SKU 기반 장바구니 전후 수량 delta 검증
- 중단된 서비스 워커 작업을 journal로 안전하게 복구하고 같은 실행의 중복 담기 방지
- 실제 장바구니 표시가격, 예상가격, 차액을 결과 화면에서 구분
- 전체 성공·부분 실패·사용자 조치 필요 상태를 명확히 분리

## 확장 프로그램

- 설치 경로는 `apps/extension` 하나로 통일
- Manifest V3 서비스 워커와 Chrome Side Panel
- content script를 Chrome이 직접 실행할 수 있는 독립 IIFE 파일로 별도 빌드
- 소스와 배포물을 하나로 통합하고, 공개 HTTPS 서버만 허용하는 단일 확장 ZIP 생성
- Apple의 절제된 계층과 Toss의 명확한 단계·숫자·안심 UX를 참고한 독자 UI
- 일반 사용자 화면에서 서버 URL, MCP, token, raw log 등 개발자 정보 제거
- 실제 장바구니 바로가기, 실패 상품 재시도, 상품 페이지 열기 제공

## ChatGPT 앱·서버

- OpenAI Apps SDK/MCP 도구와 React 위젯
- Cloudflare Worker와 Durable Object 기반의 고정 HTTPS 공개 MCP·페어링·handoff 서버
- 일반 사용자는 로컬 서버, 터널 또는 OpenAI API 키 없이 공개 앱과 확장 프로그램만 사용
- OpenAI Secure MCP Tunnel은 개발자가 로컬 MCP를 점검할 때만 사용하는 보조 경로
- 일회용 페어링 코드, device token, TTL handoff, idempotency key
- 서버 재시작에도 복원되는 로컬 상태 저장소와 비밀정보 해시 저장
- 연결 해제 시 device token·grant·pairing·handoff 일괄 폐기
- 클라이언트별·코드별 페어링 시도 제한
- 확장 프로그램의 파트너스 검색은 GPT 페어링과 독립적으로 자동 device 등록

## 쿠팡 파트너스

- 공식 HMAC 서명, Product Search, Deep Link adapter
- API 키는 서버 `.env`에만 저장
- API 미설정·미지원·오류 시 일반 쿠팡 검색과 일반 링크로 fallback
- 사용자 opt-in과 제휴 고지 전제
- 공개 확장 패키지는 직접적 사용자 혜택과 플랫폼 허용이 검증될 때까지 제휴 기능을 빌드 시점에 강제 비활성화
- 수수료 발생을 보장하지 않음

## 이번 안정화에서 달라진 점

- 정확히 일치하지 않아도 모든 유사 후보를 펼쳐 직접 선택할 수 있고, 후보가 없으면 행 안에서 검색어를 고쳐 다시 찾을 수 있음
- `일치 후보 없음` 대신 `후보 N개 보기`를 표시하고 첫 확인 필요 상품을 자동으로 펼쳐 사용자가 다음 행동을 바로 알 수 있음
- 쿠팡 검색 상단의 `최근 본 상품` 위젯을 검색 후보에서 제외해 실제 검색 결과가 후보 슬롯에서 밀리던 문제를 제거
- 한글 `리터·밀리리터·그램·킬로그램`을 규격으로 인식하고, 오타 검색 `샘다수 2리터`에서도 쿠팡이 반환한 `삼다수 2L` 후보를 직접 고를 수 있음
- 검색 결과의 아무 상품이나 고르지 않고 브랜드·핵심 제품명·용량·함량·포장 규격·묶음이 정확한 후보만 자동 선택
- 쿠팡 상세페이지 가격이 늦게 나타나는 경우 구매 영역의 최종가격만 제한적으로 재확인
- `2개 묶음 1세트`와 `단품 2개`를 모두 실물 2개로 일관되게 설명
- 5종 중 4종만 확인된 경우 확인된 4종·실물 6개·해당 금액을 명시하고, 전체 성공처럼 표시하지 않음
- GPT 앱 연결을 기술 설정 입력 없이 `연결하기 → 6자리 코드 입력 → 목록 받기` 흐름으로 단순화
- 기기 토큰 발급만으로 연결 완료로 오인하지 않고 서버의 실제 페어링 상태가 `connected=true`일 때만 연결 완료로 표시
- 같은 위젯 nonce의 재시도는 기존 연결 grant를 안전하게 복구하고, 다른 nonce의 재사용은 거부
- GPT 위젯은 공식 MCP Apps JSON-RPC bridge를 우선 사용하고 `window.openai`는 호환 fallback으로만 사용
- Web Store 패키징 시 운영 HTTPS 서버 원본을 명시적으로 주입할 수 있으며 localhost 권한은 제거
- 쿠팡 검색 카드의 `배송비 3,000원·6,000원`을 상품가로 잘못 읽던 문제를 제거하고 할인 후 판매가만 사용
- 상품과 무관한 헤더 검색 카테고리를 필수 옵션으로 오인하던 문제를 제거하고 구매 영역 내부 옵션만 검사
- 상세가격이 먼저 나타나도 장바구니 버튼 렌더링까지 제한적으로 재확인
- 상세 제목에 선택된 `50mL × 2개` 옵션 정보를 결합해 1+1 상품도 정확한 SKU로 검증
- 서버 시작 시 항상 최신 소스를 빌드하고 이전 딱담아 서버를 교체하며, health의 build ID로 오래된 런타임을 식별
- GPT 위젯 내부 페어링·전송·상태·해제 도구를 앱 전용으로 숨기고 위젯 리소스 URI를 v12로 갱신
- 기존 대화가 캐시한 v11~v5 위젯 URI도 최신 위젯 내용으로 읽을 수 있게 등록해 `resources/read` 실패를 방지
- GPT 위젯의 연결 grant를 안전한 호스트 상태와 로컬 앱 저장소에 복원하고, 새 계획을 연결된 확장 프로그램으로 자동 전송
- 완료 화면에서 `쿠팡 장바구니 보기`와 `새 목록 담기`를 별도 동작으로 제공하고 자동 탭 전환으로 인한 화면 튐을 제거
- ChatGPT 테마와 운영체제 색상 설정을 따르는 라이트·다크 디자인 토큰 적용
- 테스트 빌드는 프로젝트 공용 `output/extension-test-dist`, 사용자 설치 빌드는 `apps/extension/dist`로 완전히 분리해 테스트 실행이 설치된 확장 프로그램을 덮어쓰지 않도록 수정
- Windows 설치 스크립트가 항상 최신 소스를 프로덕션 모드로 다시 빌드한 뒤 설치 경로를 열도록 수정
- 실제 MV3 확장 프로그램과 공개 MCP 사이에서 6자리 페어링, 5종·실물 7개 목록 수신, ACK, 연결 해제 후 토큰 폐기까지 자동 E2E 검증
- 실제 Side Panel에서 상품 제외·재포함, 5종 상세검증, 장바구니 추가, 합계 표시, 장바구니 열기, 새 목록 시작까지 한 흐름으로 검증

고정 HTTPS Worker와 공개 `/mcp`는 `https://ddakdama.ddakdama.workers.dev`에 배포되어 ChatGPT v12 위젯과 MV3 확장 프로그램의 실연결 검증을 통과했습니다. 승인된 쿠팡 파트너스 키 호출과 로그인된 사용자 Chrome에서의 실제 쿠팡 장바구니 변경은 계정·권한 및 명시적 승인 후 최종 검증해야 합니다.
